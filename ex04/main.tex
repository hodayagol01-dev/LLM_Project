\section{Introduction}
This document outlines the architecture for Assignment 04, focusing on autonomous document generation. By integrating LaTeX with agentic workflows, we aim to bridge the gap between structured technical writing and intelligent automation.

\section{Assignment Context}
The objective is to establish an end-to-end pipeline capable of generating academic-grade PDFs. This requires not only typesetting capabilities but also logical control over document structure and content verification.

\section{Technical Foundation}
\subsection{LuaLaTeX and CLS Templates}
We leverage LuaLaTeX for its advanced scripting capabilities, allowing for programmatic manipulation of the document model. The project utilizes a custom \texttt{custom-academic-template.cls} to ensure stylistic consistency and adherence to academic formatting standards \cite{latex_companion}.

\subsection{Agent Orchestration and Skills}
Content generation is mediated by an orchestrator agent that delegates specialized tasks to defined ``skills''. Skills act as modular agents, ensuring clean separation of concerns between content creation, compilation, and quality assurance.

\section{Methodology}
\subsection{QA Workflow}
Our QA process employs a multi-faceted approach, checking document structure, citation integrity, and language quality. This ensures that the generated output meets the rigors of formal documentation.

\section{Repository Structure}
The project follows a modular organization:
\begin{itemize}
    \item \texttt{src/}: Core logic (orchestrator, compilation, QA).
    \item \texttt{skills/}: Declarative agent capabilities.
    \item \texttt{assets/}: External resources (images, graphs).
\end{itemize}

\section{Reproducibility, Limitations, and Future Work}
While the current implementation provides a robust skeleton, reproducibility relies on specific environment configurations.
